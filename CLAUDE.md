## Regra obrigatória — PROGRESSO.md

Sempre que fizer mudanças no código (bugfix, feature, refactor, deploy), **antes de encerrar a resposta** você deve atualizar o arquivo `PROGRESSO.md`:

- Adicionar uma linha na tabela **Bugs Corrigidos** se foi uma correção
- Adicionar uma linha na tabela **Histórico de Sessões** se foi uma sessão de desenvolvimento
- Manter o formato já existente no arquivo (tabela markdown com Data | O que foi feito)

Esta regra se aplica a toda e qualquer conversa neste projeto. Não é opcional.
